# Quick Task: CSO 보안 감사 기반 보안 수정

CSO 감사에서 발견된 5개 항목을 앱의 핵심 제약조건(B2G 1회성 납품, 정적 빌드, 익명 학생 설계)에 맞게 수정.

## Task 1: Supabase RLS 정책 강화 SQL 수정

**files:** `docs/supabase/patch_rls_fix.sql`
**action:** anon 전체 CRUD를 제거하고, licenses는 인증된 교사만 접근 가능하도록, teacher_profiles/student_profiles INSERT는 제한 조건 추가
**verify:** SQL 파일이 auth.uid() 기반 정책으로 변경되었는지 확인
**done:** RLS 정책이 anon 전체 접근을 허용하지 않음

## Task 2: Admin PIN 개선 (6자리 + crypto 기반 라이선스 키)

**files:** `src/app/admin/page.tsx`
**action:**
- PIN 기본값을 6자리로 변경 (예: "000000")
- PIN 입력 UI를 6자리 대응으로 수정
- `generateLicenseKey()`의 `Math.random()`을 `crypto.getRandomValues()`로 교체
**verify:** PIN 검증 로직이 6자리, 라이선스 키 생성이 crypto 기반인지 확인
**done:** Admin PIN 6자리, 라이선스 키 암호학적 랜덤

## Task 3: 클래스 코드 생성 crypto.getRandomValues() 적용

**files:** `src/lib/supabaseClient.ts`
**action:** `generateClassCode()` 함수의 `Math.random()`을 `crypto.getRandomValues()`로 교체
**verify:** Math.random이 더 이상 사용되지 않는지 확인
**done:** 클래스 코드가 암호학적으로 안전한 랜덤으로 생성됨

## Task 4: 개인정보 미수집 고지문 페이지 추가

**files:** `src/app/privacy/page.tsx` (신규)
**action:** "이 앱은 개인정보를 수집하지 않습니다" 고지문 페이지 생성. 앱 디자인 시스템(Fredoka 폰트, sky blue 배경, 둥근 카드) 준수
**verify:** /privacy 경로에 고지문 페이지가 렌더링되는지 확인
**done:** 개인정보 미수집 고지문 페이지 존재

## Task 5: npm audit fix (의존성 취약점 패치)

**files:** `package.json`, `package-lock.json`
**action:** `npm audit fix` 실행하여 jsPDF, flatted, picomatch, tar 등 취약점 패치
**verify:** `npm audit` 결과에서 critical/high 감소 확인
**done:** 주요 취약점 패치 완료
