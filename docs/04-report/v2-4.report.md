## 6. Lessons Learned & Retrospective

### 6.1 What Went Well

1. **Clean Separation of Concerns**: Audio processing logic (`audioAssessment.ts`) is completely decoupled from UI rendering (`AudioVisualizer.tsx`). Makes both easier to test, modify, and integrate independently.

2. **Stub-First Design**: Clearly marked `[STUB]` placeholders with TODO comments make it obvious where Wasm modules will replace dummy logic. Interim behavior (energy correlation) is reasonable for testing without Wasm dependency.

3. **Starter Architecture Alignment**: Co-locating AudioVisualizer in `src/app/lesson/[unitId]/` follows the Starter-level pattern of feature-scoped components. Direct `@/lib/` imports from components are established convention here.

4. **Comprehensive Real-Time UX**: Volume indicator + waveform canvas + score gauge + phoneme bars provide rich visual feedback during recording, critical for young learners.

5. **TypeScript Rigor**: All public functions, callbacks, and component props are fully typed. No implicit `any` types except for necessary webkit compat.

### 6.2 What Needs Improvement

1. **LessonClient Integration Deferred**: AudioVisualizer is ready but not wired into any lesson step. Should have been included in this round or marked as a required follow-up.

2. **Wasm Module Absence**: Stubs are reasonable for MVP, but without real MFCC/DTW, accuracy scores are dummy values. Production use requires Wasm completion.

3. **Limited Test Coverage**: No unit tests for audio buffer edge cases, permission denial flows, or score calculation logic. Recommend adding before production.

4. **Browser Compatibility Note**: Relies on ScriptProcessorNode (deprecated since 2017). Works today but should migrate to AudioWorklet for future-proofing.

### 6.3 What to Try Next

1. **Wasm Module Implementation**: Create Wasm stubs for MFCC (13 coefficients per frame) and DTW (dynamic time warping). Can be done in parallel with other v2 features.

2. **Reference Audio Database**: Pre-compute MFCC for all ~300 words using existing TTS audio files. Store in IndexedDB for fast similarity lookup.

3. **AudioWorklet Migration**: Modernize audio buffer handling. Can keep ScriptProcessorNode as fallback.

4. **Unit Tests**: Add Jest tests for assessment edge cases (short buffer, no permission, assessment timeout, etc.).

5. **Integration into Lesson Flow**: Add `say_and_check_advanced` step or enhance existing `say_and_check` step to use AudioVisualizer for pronunciation feedback.
