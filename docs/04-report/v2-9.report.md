## 8. Lessons Learned & Retrospective

### 8.1 What Went Well (Keep)

1. **Graceful Degradation Pattern**: Missing image fallback (onError → null) ensures UX is never broken even if asset generation lags. This is production-ready.

2. **Reusable Component Design**: Single `WordImage` component with size variants (sm/md/lg) and optional animation achieves high code reuse across 5 lesson steps. No duplication.

3. **Parallel Asset Generation**: 4 scripts provide options (Gemini primary, Imagen alternative, SVG fallback, verification). Adds robustness without blocking implementation.

4. **Spring Animation Excellence**: Framer Motion physics (stiffness:300, damping:22) creates kid-friendly pop-in effect. Tested across multiple contexts.

5. **Data-Driven Naming**: Image file naming convention (`${wordId}.png`) maps directly to curriculum word IDs, making maintenance trivial and preventing conflicts.

### 8.2 What Needs Improvement (Problem)

1. **Asset Generation Scope Creep**: Design requested "300-word illustrations" but 4 scripts, 2 APIs, and 3 fallback strategies were implemented. While excellent, this was more than scope — should have clarified ownership boundary earlier.

2. **Partial Asset Coverage**: ~200 of 300 images present at completion. While graceful fallback handles this, the feature feels incomplete visually. Need batch generation completion before production.

3. **BlendTapStep Inconsistency**: Uses inline image implementation instead of shared `WordImage` component. Creates minor maintenance burden (2 animation param sets: damping:22 vs damping:20).

4. **Import Order Violation**: framer-motion and lucide-react appear after @/ imports in LessonClient.tsx. This is a persistent issue since Round 7 — should have been fixed in earlier rounds.

5. **Accessibility Oversight**: WordImage component missing `role="img"` on container and dedicated `aria-label`. WCAG A compliance could be improved.

### 8.3 What to Try Next (Try)

1. **Asset Completion Script**: Create `npm run generate-missing-images` command that auto-detects gaps and fills them. Use skip-existing optimization already in place.

2. **Refactor BlendTapStep Image**: Extract inline image code to use shared `WordImage` component. Normalize animation params across all uses (standardize on damping:22).

3. **Fix Persistent Import Order**: Make import order check a pre-commit hook. Catch violations before they reach code review.

4. **Enhance Accessibility**: Add `role="img"` and `aria-label={alt}` to WordImage. Run Lighthouse a11y audit.

5. **Document Asset Pipeline**: Create Asset Generation Guide (docs/guides/image-generation.md) documenting Gemini → PNG workflow, caching strategy, and retry logic.
