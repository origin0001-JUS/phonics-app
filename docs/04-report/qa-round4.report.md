## 3. 상세 결과

### 3.1 Part O: Word Family 범위 오류

**문제**: 가족 진행 텍스트가 범위를 초과하여 표시됨 (예: "Family 4 / 3"). 마지막 단어 완성 후 UI 멈춤.

**해결**:
- **safeFamilyIdx 클램핑**: `Math.min(familyIdx, Math.max(0, families.length - 1))`로 인덱스가 배열 범위를 초과하지 않도록 보장
- **탭 가드**: `const [tapping, setTapping] = useState(false)`로 상태 전환 중 중복 탭 방지
- **경계 검사**: 진행 표시(L262)와 다음 가족 이동(L148) 모두 클램핑된 인덱스 사용

**지표**: 설계 일치 6/6 항목 통과 (100%)

---

### 3.2 Part P: Word Family 오디오 타이밍 및 오답 효과음

**문제**: 오답 효과음이 안정적으로 재생되지 않음. 상태 전환이 너무 빨라 단어 오디오가 잘림.

**해결**:
- **playSFX('wrong') try/catch 처리**: L113에서 오디오 오류가 인터랙션을 깨지 않도록 래핑
- **상태 전환 전 1400ms 지연**: 600ms에서 증가하여 단어 오디오 전체 재생 보장
- **오디오 시퀀스**: 단어 오디오 200ms → 정답 효과음 1400ms로 적절한 레이어링

**지표**: 설계 일치 3/3 항목 통과 (100%). 오디오 타이밍 600ms → 1400ms (+800ms 버퍼)

---

### 3.3 Part Q: Say&Check 자동 재생

**문제**: Say & Check 단계 진입 시 또는 단어 변경 시 단어 오디오가 자동 재생되지 않음.

**해결**:
- **idx 변경 시 useEffect**: L937-950에서 단어 인덱스 변경을 감지하여 자동 재생 트리거
- **TTS 전 300ms 지연**: 브라우저 자동 재생 정책 준수 및 오디오 컨텍스트 초기화 시간 확보
- **hasListened 잠금 해제**: 1500ms 후 `true`로 설정하여 마이크 버튼 자동 활성화
- **상태 초기화**: 단어 변경 시 `hasListened`와 `result` 초기화

**지표**: 설계 일치 6/6 항목 통과 (100%). 마이크 버튼이 수동 조작 없이 자동 활성화.

---

### 3.4 Part S: 발음 정확도 UI

**문제**: STT 평가가 시각적 정확도 점수 없이 무음으로 통과/불통과 처리됨.

**해결**:
- **정확도 퍼센트**: `Math.round((result.confidence || 0) * 100)%` 피드백 하단에 표시 (L1059)
- **진행 바**: 신뢰도 기반 시각적 바 표시 (L1061-1065)
- **조건부 색상**: 일치 시 녹색, 불일치 시 주황색 (L1063)

**지표**: 설계 일치 5/5 항목 통과 (100%). UI 요소 2개 추가 (정확도 텍스트 + 진행 바)

---

### 3.5 Part T: 복습 큐 SRS 날짜 오버라이드

**문제**: 틀린 답이 복습 큐에 즉시 나타나지 않음. `calculateNextReview(srsCard, 0)`가 `nextReviewDate = 오늘 + 1일`로 설정하여 내일까지 복습 대상에서 제외됨.

**해결**:
- **nextReviewDate 오버라이드**: L278에서 틀린 답에 대해 `updated.nextReviewDate = today`로 명시적 설정
- **즉시 저장**: Dexie에 비동기 즉시 기록 (L279)
- **정답 여부 게이트**: `!correct`인 경우에만 오버라이드 적용 (L267)
- **에러 처리**: try/catch로 비동기 작업 오류가 수업 흐름을 깨지 않도록 보호

**지표**: 설계 일치 6/6 항목 통과 (100%). 틀린 답이 복습 배지와 큐에 즉시 반영.

## 7. 교훈

### 7.1 잘된 점
1. **명확한 문제 명세**: 각 버그에 명시적 근본 원인과 해결책이 계획서에 포함되어 구현 시 모호함 제거
2. **국소적 수정**: WordFamilyBuilder와 SayCheckStep이 버그가 발생한 파일에 공존하여 수정과 검증이 용이
3. **데이터 기반 UI 개선**: STT 엔진의 `result.confidence`를 직접 사용하여 추가 데이터 처리 없이 정확한 피드백 제공
4. **비동기 SRS 기록 패턴**: IIFE로 래핑된 비동기 Dexie 작업이 수업 흐름을 차단하지 않으면서 DB 저장 처리

### 7.2 개선할 점
1. **오디오 에러 모니터링**: playSFX의 try/catch가 오류 원인을 숨길 수 있음. 로깅 추가 고려
2. **탭 가드 패턴**: boolean 가드는 빠른 해결책. 향후 상태 머신(idle → processing → ready) 고려
3. **브라우저 자동 재생 정책**: 300ms 지연은 실용적 우회. 모바일 기기별 크로스 브라우저 테스트 필요
