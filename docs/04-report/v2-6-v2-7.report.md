## 5. Implementation Details

### 5.1 V2-6 Implementation Summary

#### 5.1.1 Phoneme Weakness Analysis

**Location**: `src/lib/exportReport.ts:63-120`

**Implementation**:
- `PHONEME_LABELS`: 23 IPA→display name mappings (design spec + 2 beneficial additions for L3/L4)
- `analyzePhonemeWeakness(allCards, allWords)`:
  - Creates word ID → phonemes map from curriculum
  - Iterates all cards, marking stage≤1 or easeFactor<2.0 as weak
  - Aggregates by phoneme, calculates weaknessRate = (weakCount/totalCount)*100
  - Excludes phonemes with totalCount<2 (insufficient data)
  - Returns top 10 sorted by weaknessRate descending

**Data Integrity**: All 23 phoneme labels verified against design Section 2.2. Algorithm matches design Section 2.2.

#### 5.1.2 Weekly Statistics

**Location**: `src/lib/exportReport.ts:124-164`

**Implementation**:
- `calculateWeeklyStats(allLogs, allCards, weeksBack=4)`:
  - Calculates 4-week trailing window (Mon–Sun boundaries)
  - Groups logs by week, sums totalMinutes + sessionCount
  - Approximates wordsLearned via nextReviewDate proximity to week range
  - Returns `WeeklyStats[]` in chronological order

**Data Source**: ActivityLog[] (durationMinutes, date) + VocabularyCard[] (nextReviewDate).

#### 5.1.3 PDF Export

**Location**: `src/lib/exportReport.ts:378-416`

**Implementation**:
- `generatePDF(report)`: async function using dynamic imports
  - Captures `#report-content` DOM with html2canvas (scale: 2 for retina)
  - Creates A4 PDF with 10mm margins via jspdf
  - Adds image with automatic page splitting on overflow
  - Filename: `phonics300_report_{studentName}_{YYYY-MM-DD}.pdf`
  - Uses anchor element for download trigger

**Bundle Optimization**: Dynamic imports ensure PDF/canvas code only loads when used.

#### 5.1.4 CSV Enhancement

## 7. Lessons Learned & Retrospective

### 7.1 What Went Well (Keep)

1. **Data-Driven Design**: Using JSON-informed word lists (phonemes, onset/rime, wordFamily) from day one ensured 100% data integrity. No rework needed.

2. **Design Compliance**: 98% match rate on first pass indicates excellent plan→design→implementation alignment. Zero iterations needed.

3. **Prefix Strategy for ID Conflicts**: Using `l3_` and `l4_` prefixes for new words cleanly avoided 10+ duplicate ID conflicts that would have required design changes. Follows design Section 3.8 intentionally.

4. **Dynamic Import Strategy**: Recharts + jspdf dynamic imports keep vendor bundle light until needed. Report page loads fast; charts only hydrate on visit.

5. **Pedagogical Soundness**: L3/L4 units follow established phonics progression (consonant clusters → digraphs → advanced vowels). All words are authentic English vocabulary for Korean children's level.

### 7.2 What Needs Improvement (Problem)

1. **Chart Color Coding in Design**: Design specified bar colors but didn't define exact hex values. Implementation had to infer from design intent (green/amber/red 3-tier). Minor issue; implementation is sensible.

2. **microReading Translation Quality**: Korean translations are hand-written; no bilingual review process was documented. Should have a checklist for accuracy.

3. **Word Image Assets Not Linked**: Design mentions `imagePath` for all 170 words, but images don't exist yet. This is a known deferred item but should be flagged prominently in next round.

4. **Bundle Size Documentation**: New dependencies add 154 KB gzipped. No explicit webpack bundle analysis was done; recommend adding bundle size monitoring.

### 7.3 What to Try Next (Try)

1. **Automated Data Validation**: Add pre-commit hook to run `merge-l3l4.ts` on curriculum.ts changes. Prevents accidental duplicate IDs.

2. **Phoneme Labels i18n**: PHONEME_LABELS is currently English-only. Next round should consider Korean IPA labels for bilingual reports.

3. **CSV Encoding Test**: Korean CSV export was tested with BOM, but no formal test on Windows Excel. Recommend E2E test in next iteration.

4. **Chart Accessibility**: Recharts BarChart/LineChart work but lack ARIA labels. Next round: add role="img" + aria-label to chart containers.

5. **Word Image Batch Integration**: For v2-polish round, set up image asset pipeline (SVG or AI-generated PNGs) and wire into MagicEStep + Blend & Tap step.
