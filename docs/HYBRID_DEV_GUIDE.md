# 안티그래비티 ↔ 클로드 코드 ↔ 클로드 웹 (GitHub 기반 하이브리드 병렬 개발 가이드)

이 문서는 1인 개발자가 회사(Claude Web)와 집(로컬 Antigravity + Claude Code)을 오가며 작업할 때, **GitHub를 중앙 허브로 삼아** 끊김 없이 병렬 개발을 진행하기 위한 궁극의 컨텍스트 동기화 워크플로우를 설명합니다.

---

## 1. 핵심 원칙: GitHub ↔ `HANDOFF.md` + `CLAUDE_TASKS.md` 동기화

모든 에이전트는 서로의 대화를 볼 수 없지만, **GitHub 저장소의 문서들과 최신 커밋 내역**을 통해 완벽하게 소통합니다.

- **`HANDOFF.md` (중앙 통제실)**: 현재 어떤 창/터미널이 무슨 작업을 하고 있는지(R&R)와 최신 진행 상황을 요약하는 매우 짧은 문서.
- **`CLAUDE_TASKS.md` (클로드 전용 백로그)**: 로컬 터미널의 Claude Code가 수행해야 할 구체적인 라운드별 코딩 지시, 파일 수정 내역, 성공 조건(빌드 통과 등)이 상세히 기록된 작업 지시서.

### 각 에이전트별 역할
- **Claude (Web)**: 회사 환경. GitHub와 직접 연동. 직접 코딩, 아키텍처 수립, 리팩터링 후 GitHub에 Push. 로컬 실행 불가.
- **Antigravity (로컬 GUI)**: 기획자 역할, 아키텍처 수립, 작업 분배(`/plan`), UI 컴포넌트 수정, 브라우저 E2E QA, 로컬 빌드 검증 전담.
- **Claude Code (로컬 CLI)**: 묵묵히 `CLAUDE_TASKS.md`에 지시된 백엔드 로직 작성 및 터미널 빌드, 대규모 스크립트 실행 전담.

**절대 규칙**: 작업을 시작하기 전에는 무조건 `git pull`을 받고, 작업이 끝나면 무조건 `git push`를 해야 합니다.

---

## 2. 🔄 3-Way Git Handover 시나리오 (회사 ↔ 집)

### Step 1. 회사에서 작업 시작 (Claude Web 로딩)
회사에서 Claude Web을 엽니다. (이미 GitHub에 권한이 연동되어 있습니다.)
> *"현재 GitHub 레포지토리의 최신 `HANDOFF.md`를 읽고 현재 진행 상황을 요약해 줘. 그리고 오늘 진행해야 할 [특정 기능] 코딩을 시작하자."*

### Step 2. 회사에서 퇴근 전 (Claude Web 작업 마무리)
Claude Web이 코딩을 모두 마쳤다면, 퇴근하기 전에 다음과 같이 지시합니다.
> *"오늘 작업한 내용들을 `HANDOFF.md`에 'Latest Updates'로 기록하고, 다음 로컬 담당자가 해야 할 일(예: API 키 연동, 로컬 빌드 테스트 등)을 적어둬. 그리고 이 모든 변경사항을 GitHub에 커밋하고 푸시(Push)해 줘."*

### Step 3. 집에서 작업 시작 (로컬 Antigravity 실행)
집에 와서 로컬 에디터를 열고 새 안티그래비티 탭에서 `/load` 단축어를 칩니다.
`/load` 워크플로우는 자동으로 `git pull`을 수행하고 `HANDOFF.md`를 읽은 뒤 현재 상황을 즉시 브리핑합니다.

### Step 4. 집에서 로컬 환경 전용 작업 진행
- 외부 연동이나 무거운 터미널 작업은 `/claude [지시]` 단축어를 통해 로컬 **Claude Code**에게 던집니다.
- 브라우저 테스팅이 필요한 UI 작업은 **Antigravity**가 직접 진행합니다.

### Step 5. 출근 전 (로컬 작업 마무리)
작업을 마치고 컴퓨터를 끄기 전 `/save` 단축어를 칩니다. 
`/save` 워크플로우는 자동으로 진행 상황을 `HANDOFF.md`에 요약하여 적고, 변경된 모든 코드를 GitHub에 푸시(Push)해 줍니다.

## 3. 안티그래비티 전용 글로벌 단축어 (Slash Commands)

이 단축어들은 전역 폴더(`~/.agents/workflows/`)에 설치되어 모든 워크스페이스에서 작동합니다. Git 동기화 로직이 내장되어 있습니다.

### 💾 `/save` (작업 중단 및 Git Push)
- **작동 방식**: 
  1. 현재 대화창의 작업 내용과 다음 할 일을 `HANDOFF.md`에 업데이트합니다.
  2. 터미널 명령어를 통해 `git add .`, `git commit -m "chore: save local context"`, `git push`를 자동으로 실행합니다.
  3. 회사(Claude Web)로 넘어갈 준비가 완료되었음을 알립니다.

### 🔄 `/load` (새 창 열기 및 Git Pull)
- **작동 방식**: 
  1. 켜지자마자 터미널을 통해 `git pull`을 실행하여 회사(Claude Web)에서 작업한 최신 코드를 당겨옵니다.
  2. `HANDOFF.md`를 읽고 "회사에서 여기까지 작업해두었네요. 이어서 로컬 작업을 진행할까요?"라고 브리핑합니다.

### 🤖 `/claude [지시내용]` (클로드 작업 할당)
- **작동 방식**: 
  클로드용 프롬프트 파일(`CLAUDE_PROMPT_xxx.md`)을 생성하고 터미널 명령어(`claude -p 파일명`)를 던져줍니다.
  *주의: 로컬 클로드의 프롬프트 파일 시스템 명령(`CLAUDE.md`)에도 작업 종료 시 무조건 Git Push를 하도록 설정되어 있습니다.*

---

## 4. 클로드 코드 (로컬 CLI) 작업 마무리 프로토콜

클로드는 터미널에서 혼자 구동된 뒤 작업이 끝나면 다음을 강제 수행합니다. (이 내용은 `CLAUDE.md`에 정의됨)

1. **테스트 및 검증**: `npm run build` 성공 여부 확인.
2. **`HANDOFF.md` 업데이트**: 작업 결과 기록.
3. **사용자 보고 및 Git Push 자동화**: 
   터미널에서 자동으로 `git add`, `commit`, `push`를 한 뒤 사용자에게 텍스트로 알립니다.
   > *"✅ 작업이 성공하고 GitHub에 푸시되었습니다. 안티그래비티 대화창으로 돌아가서 QA를 요청하세요."*
