# Claude Code Bug Fix Tasks (QA Round 2)

이 문서는 QA 1차 테스트 결과와 이전 작업(v2-bugfix)의 결과를 토대로, 클로드가 이어서 진행해야 할 필수 로직 버그 수정 지시서입니다.

## 클로드 코드 필수 수정 과제

### 1. 데이터 저장 및 SRS 연동 로직 (가장 시급함 - 이전 Bug #7)
이전 분석(`v2-bugfix.analysis.md`)에서 해당 이슈가 이미 Round 13-D에서 수정되었다고 판단하여(DEFERRED) 스킵하였으나, 실제 로컬 브라우저로 QA를 진행해 본 결과 **여전히 심각한 버그가 발생 중**입니다. 반드시 코드를 열고 아래 현상을 수정해야 합니다.
- **버그 1 (진행도 휘발):** 유닛 레슨을 진행하던 도중(예: stage 5/8) 홈 화면으로 나갔다가 다시 유닛으로 들어오면, 중간 진행 상황이 모두 날아가고 처음(stage 1/8)부터 다시 시작됩니다.
- **버그 2 (SRS 리뷰 미반영):** 퀴즈 진행 중 오답을 입력했음에도, 홈 화면의 Review(복습) 큐 배지가 즉시 늘어나지 않고 `All caught up!` 메시지가 그대로입니다.
- **요구사항:** 레슨 도중 이탈하더라도 저장된 데이터(`sessionStorage` 또는 `IndexedDB`)를 활용하여 정확히 진행하던 스테이지부터 재개되도록 세션 저장 로직을 보강해야 합니다. 또한, 퀴즈 오답 데이터가 즉시 SRS 복습 큐 스토리지에 Push되도록 연동 로직을 고쳐주세요.

### 2. 세션 복구 상세도 개선 (이전 Bug #4)
이전 작업에서 보류되었던(DEFERRED) 세부 세션 복구 기능을 이번 스텝에서 구현합니다.
- **버그:** 현재 `stepIndex` (예: 2/3단계) 레벨까지만 세션 복구가 되기 때문에, 특정 퀴즈 내의 3번째 문제를 풀다 나가도 다시 그 단계의 1번째 문제로 돌아갑니다.
- **요구사항:** `sessionStorage`의 스키마를 확장하여 `subStepIndex` (word index, quiz index 등)를 포함하여 저장하고 복원하도록 코드를 수정해 주세요. 이를 통해 사용자가 이탈한 정확한 '문제' 단위에서 재개할 수 있어야 합니다.
