# Phonics 300 - V2 마스터 종합 진행 상황 보고서

> **작성일**: 2026-03-17
> **목적**: 유실된 이전 대화(가족 베타테스트 및 QA 라운드)의 컨텍스트를 복원하고, Antigravity, Claude(Web), Claude Code 간의 유기적인 R&R, Vercel 배포 현황, 그리고 최신 Round 14(AI 립싱크) 진행 상황까지 프로젝트 전체를 총망라하여 요약합니다.

---

## 1. 프로젝트 종합 개요 (Executive Summary)
**Phonics 300**은 V1.5의 기본 파닉스 학습 기능을 넘어, 시각적 직관성(Track A), 코어 인터랙션 게임화(Track B), 그리고 고도화된 평가/확장 인프라(Track C)를 갖춘 **V2**로 성공적으로 진화했습니다. 
수백 개의 에셋(단어 이미지, 음소 오디오, 스토리 만화, 립싱크 영상 등)이 자동화 스크립트를 통해 생성되었으며, 가족 단위의 자체 베타 테스트와 여러 라운드에 걸친 꼼꼼한 QA를 거쳐 현재 **Vercel에 성공적으로 프로덕션 배포**되어 있는 상태입니다.

---

## 2. 개발 환경 및 역할 분담 (Hybrid R&R)
프로젝트의 효율성을 극대화하기 위해 다중 환경과 AI 에이전트들이 유기적으로 협력하는 구조를 확립했습니다.

### 🧑‍💻 사용자 (User) & 가족 대상 베타 테스터
- **역할**: 프로젝트 총괄 지휘, Vercel 배포 관리, 외부 API 결제 및 키 관리, **가족 대상 실제 앱 베타 테스트 진행 및 피드백 수집**.
- **기여**: 실제 아이들/가족들이 앱을 사용하며 느낀 UX적 불편함(Blend & Tap 오디오 딜레이, 음소 단위 발음의 부정확성 등)을 캐치하여 QA 라운드의 핵심 과제(Round 1~13)로 제시함.

### 🤖 Antigravity (로컬 IDE 에이전트)
- **환경**: 사용자의 로컬 노트북 (VS Code / 터미널 제어 권한 보유)
- **역할**: 핵심 UI/UX 로직 코딩, 상태 관리(Zustand), 컴포넌트 설계, 로컬 빌드 검증 및 테스트.
- **주요 수행**: `LessonClient.tsx` 플로우 제어, `MouthVisualizer.tsx` 컴포넌트 구현, 오디오/비디오 시퀀서 동기화 등 앱의 "뼈대와 근육" 완성.

### 💻 Claude Code (CLI 에이전트)
- **환경**: 사용자의 로컬 터미널 내 백그라운드 실행
- **역할**: 대규모 에셋 생성 스크립트 작성 및 묶음 실행, 파일 시스템 전수 조사.
- **주요 수행**: ElevenLabs를 통한 170개 음소 오디오 일괄 생성, `curriculum.ts`와 파일 시스템 교차 검증을 통한 누락 이미지 보완.

### 🌐 Claude (Web 에이전트 - 회사 환경)
- **환경**: 회사 웹 브라우저 (로컬 실행/외부 API 접근 불가)
- **역할**: 높은 지능을 바탕으로 한 구조적 리서치, 문서화, AI 파이프라인(립싱크 API 모델 등) 비교 분석 및 아키텍처 수립.
- **주요 수행**: `HANDOFF.md` 작성, 발음 참조 데이터(`pronunciationGuide.ts`) 구축, VEED Fabric 등 최적의 API 솔루션 탐색 및 스크립트 작성 (실행은 자택 로컬로 이관).

## 3. 베타 테스트 및 멀티 라운드 QA 히스토리 (컨텍스트 복원)
대화창 유실로 흩어졌던 **"가족 베타 테스트 기반의 정밀 QA 및 수정 내역"**을 아래와 같이 복원 및 정리합니다. 이 과정이 V2의 완성도를 극적으로 높였습니다.

*   **QA 피드백 1: "발음이 너무 겹치고 빨리 지나가요."**
    *   **해결 (Round 13)**: `LessonClient.tsx`의 Blend & Tap 스텝에서 전체 단어 발음 전 1.8초(1800ms)의 딜레이를 추가하여 아이들이 각 음소를 인지할 여유를 확보함.
*   **QA 피드백 2: "알파벳 이름(Letter Name)으로 발음해서 헷갈려요."**
    *   **해결**: 브라우저 기본 TTS 결함을 파악하고, Claude Code를 가동해 ElevenLabs로 고품질의 순수 음소(Onset/Rime) 오디오 170개를 `public/assets/audio/phonemes`에 생성 후 적용.
*   **QA 피드백 3: "단어 그림이 너무 작고, 어떤 건 안 나와요."**
    *   **해결**: 이미지 크기를 +30~40% 확대하는 UI 수정. 전수 조사를 통해 누락된 815개 이미지를 완벽히 채워넣음 (현재 Missing 0건).
*   **QA 피드백 4: "처음 화면에서 Foxy가 어색해요."**
    *   **해결 (V2-8/9)**: 오디오 전용 플레이어에서 `Foxy_english.mp4` 비디오 시퀀서로 변경. 재생 끝부분의 오디오 튀는 현상을 막기 위해 영상 마지막 1.0초를 조기 커팅(onTimeUpdate)하도록 안정화.

## 4. 지금까지의 주요 개발 성과 (V2 Core Features)
V2 목표였던 Track A, B, C가 모두 성공적으로 코드베이스에 안착, **Vercel에 배포**되었습니다.

*   **Track A (UI 직관성)**: 3D 캐릭터 온보딩, 이중언어 비디오 시퀀서 홈 화면, 발음 듀얼 뷰(Mouth + Cross Section), 300단어 팝업 이미지.
*   **Track B (인터랙션)**: Magic e 드래그 퀴즈, Word Family Onset 조합 미니게임, Decodable Story 만화 슬라이드.
*   **Track C (인프라)**: Wasm 기반 실시간 AI 발음 평가 엔진(파형 시각화), L3/L4 확장(자음군/이중모음), Recharts 기반 학부모용 취약 음소 PDF 리포트.

## 5. 최신 진행 상황: Round 14 (AI 립싱크 아바타 통합)
현재 프로젝트는 **웹 Claude가 기획하고 로컬 Antigravity/Claude Code가 실행**하는 핸드오프 체제로 "Round 14: AI 립싱크"의 첫 단계를 밟고 있습니다.

1.  **완료된 준비 (Task 14-A)**
    *   `representativeWords.ts`에 92개 주요 단어 및 Sound Focus 타겟 정리 완료.
    *   웹 Claude를 통해 `pronunciationGuide.ts` (발음 난이도/혼동상 비교) 설계 완료.
    *   기존 Flow AI 수동 작업을 대체할 **VEED Fabric 1.0 스크립트(`scripts/test-veed-fabric-sample.ts`)** 작성 완료.
2.  **현재 블로커 및 넥스트 스텝 (가정/로컬 환경 즉시 실행 요망)**
    *   `HANDOFF.md`에 명시된 대로, 로컬 환경에서 `.env.local`에 API 키(FAL, ElevenLabs)를 세팅하고 `npm run test-fabric-sample`을 실행하여 3개의 샘플 영상을 추출해야 합니다.
    *   샘플 품질 판단이 "OK"로 나면, 즉시 전체 109개 립싱크 비디오를 일괄 생성하고 앱(`MouthVisualizer.tsx`)에 연동하는 코딩 스텝(Task 14-B, C)으로 넘어갑니다.

## 요약 및 결론
**Phonics 300 V2**는 기획부터 베타버전, 멀티 라운드 QA, 그리고 Vercel 실서버 배포까지 매우 모범적인 "AI 에이전트 협업 & 사용자 주도 개발" 파이프라인을 타고 순항 중입니다. 
유실되었던 소중한 가족 테스트 피드백과 QA 히스토리도 본 문서에 완전히 영속화되었으며, 이제 남은 마지막 화룡점정인 **AI 립싱크 아바타(Round 14)** 연결만 마무리되면 궁극의 MVP가 완성됩니다.
