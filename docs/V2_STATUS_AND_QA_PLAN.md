## 2. 2차 종합 QA 테스트 체크리스트 (기능 및 품질 확인 리스트)

이번 QA는 모든 기능이 유기적으로 결합되어 정상 작동하는지, UX/UI 품질이 우수한지를 중점적으로 검사합니다.

### A. 온보딩 및 홈 화면 품질 (V2-8, 10)
- [ ] 첫 진입 시 Foxy 마스코트 이미지가 정상적으로(새벽, 선명하게) 렌더링되는가?
- [ ] 영어 "Hi I'm Foxy!"와 한국어 안내가 끊김이나 지연(Delay) 없이 매끄럽게 재생되는가?
- [ ] 음성 재생 상태(재생 중/대기 중)에 따라 Foxy의 입모양/애니메이션 상태가 정확히 변하는가?

### B. 시각적 단어 학습 및 UI (V2-9)
- [ ] Blend & Tap 완료 후 단어 이미지가 어색하지 않게 부드러운 애니메이션(Scale-in)으로 나타나는가?
- [ ] 1차 QA 피드백대로 단어 이미지가 충분한 크기(+30%)로 보이며 모바일 기기에서도 식별하기 좋은가?
- [ ] 이미지가 누락된 단어(404 에러 등)의 경우 Fallback 글자 박스 UI가 이상 없이 노출되는가?

### C. 코어 인터랙션 (Magic E, Word Family, Story) (V2-1, 2, 3)
- [ ] **Magic e**: 드래그 앤 드롭이 부드럽게 작동하며, 'e'가 붙었을 때 이미지 쌍과 오디오(단모음->장모음)가 정상적으로 바뀌는가?
- [ ] **Word Family**: 주어진 라임(예: '-at')에 Onset을 조합했을 때 단어 생성이 직관적이고 애니메이션이 끊기지 않는가?
- [ ] **Decodable Story**: 슬라이드/만화 패널 넘김 시 음성(TTS)이 겹치지 않고 타이밍에 맞게 재생 및 하이라이트 되는가?

### D. AI 발음 평가 및 학습 도구 (V2-4, 5, 6, 7)
- [ ] **발음 평가**: 마이크 권한 요청이 정상 작동하고, 발음을 했을 때 파형 UI가 즉각 반응하며 점수가 산출되는가?
- [ ] **대시보드 (오프라인 모드)**: 오프라인 IndexedDB에 저장된 학습 데이터가 구조적으로 잘 묶여있는가? (DB 연결 에러 시 Graceful degradation이 일어나는가?)
- [ ] **리포트 내보내기**: 취약 음소 통계가 정확히 반영되며, PDF/CSV 다운로드가 에러 없이 동작하는가?
- [ ] **L3/L4 확장**: 신규 유닛(Unit 25 이상)이 보이며 잠금 해제 조건이 정상적으로 동작하는가?

## 3. 기능별 다중 QA 테스트 케이스 (다양한 시나리오)

아래 테스트 케이스들을 통해 예외 상황에서도 시스템이 견고한지 확인합니다.

### Test Suite 1: 온보딩 트랜지션 및 오디오 동기화 페일오버
*   **TC 1.1 (정상 플로우)**: 앱 초기화 (IndexedDB 삭제 후) 후 첫화면 진입 시, 영어 재생 -> 즉시 한국어 재생 확인. 애니메이션도 Talking -> Idle로 자동 전환됨을 확인.
*   **TC 1.2 (가벼운 인터럽트 플로우)**: 오디오가 재생 중일 때 화면 빈 공간이나 다른 버튼을 연타하여 다른 페이지(예: 레벨 선택)로 강제 이동 시, 이전 화면의 오디오가 계속 재생되는지(중단되어야 정상) 확인.

### Test Suite 2: 단어 이미지 에셋 예외 처리
*   **TC 2.1 (이미지 캐싱 로직)**: Blend/Tap 스테이지 진입 후 한 번 로드된 이미지를 다시 확인할 때 로딩 딜레이(레이아웃 시프트) 없이 바로 뜨는지 확인.
*   **TC 2.2 (Fallback 텍스트 노출)**: 존재하지 않는 단어(Dummy Word)를 주입하거나 네트워크 연결 오프라인 후(혹 파일명 의도적 오입력) 이미지 로드 실패 시, 앱이 다운되지 않고 글자만 있는 Fallback UI를 잘 출력하는지 확인.

### Test Suite 3: Magic e & Word Family 인터랙션 충돌
*   **TC 3.1 (Magic e 오터치/빠른 터치)**: 'e' 타일을 지정된 위치가 아닌 엉뚱한 화면 구석으로 드래그 후 놨을 때 원래 자리로 튕겨져 복귀하는지 확인. 
*   **TC 3.2 (Word Family 연타)**: Onset 버튼 'b', 'c', 'h' 등을 매우 빠르게 따다닥 연타했을 때 애니메이션이나 TTS 재생 큐가 꼬여서 음성이 여러개 겹쳐 들리지 않는지(재생 중단 및 갱신 혹은 큐잉 처리) 확인.

### Test Suite 4: Decodable Story 슬라이드 제어
*   **TC 4.1 (음성 중첩 확인)**: TTS가 문장을 읽고 있는 도중에 사용자가 '다음 패널' 스와이프를 강제로 넘겼을 때, 이전 문장 음성이 즉시 멈추고 새 문장의 음성이 시작되는지 확인.
*   **TC 4.2 (완독 처리)**: 6개의 패널을 끝까지 넘기고 마지막 모달까지 확인한 직후, 홈 화면(Unit View)으로 돌아왔을 때 해당 스토리 뱃지가 '완료(Completed)' 스탬프로 정확히 바뀌는지 확인.

### Test Suite 5: AI 평가 엔진 및 마이크 예외 상황
*   **TC 5.1 (권한 거부 대응)**: 권한 요청 알림 시 '차단(Block)'을 눌렀을 때 오류 팝업이나 "설정에서 마이크를 켜주세요" 등의 안내 UI가 친절하게 제공되며 하얀 화면(White Screen) 에러로 빠지지 않는지 확인.
*   **TC 5.2 (과도한 소음/무음 발성)**: 백그라운드 소음이 너무 클 때나 사용자가 아무 말도 하지 않았을 때, Wasm 모듈이 뻗지 않고 "0점" 또는 "다시 시도해주세요" 처리로 반환하는지 확인.

### Test Suite 6: 리포트 내보내기 및 렌더링 호환성
*   **TC 6.1 (PDF 인쇄 여백 체크)**: 데이터가 많은 학생(20개 이상 유닛 완료)의 리포트를 PDF로 Export할 때, 차트와 취약 음소 랭킹이 다음 페이지로 넘어갈 때 잘리거나 텍스트가 깨지지 않는지 확인.
*   **TC 6.2 (모바일 해상도 대응)**: 취약 음소 차트(Recharts)가 세로 모드(모바일 폭)에서 삐져나가지 않고 X/Y축 레이블이 식별 가능하게 축소/정렬되는지 확인.
