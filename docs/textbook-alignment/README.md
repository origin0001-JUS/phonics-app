# 교재·교과서 스캔 자료 활용 가이드

> 파닉스 교재 및 초등 3·4학년 영어 교과서 스캔 자료를 앱 개발에 활용하기 위한 전략 문서

---

## 1. 핵심 원칙: 참조만, 복사는 절대 불가

| 구분 | 허용 | 금지 |
|------|------|------|
| 커리큘럼 순서 참조 | O | - |
| 단어 빈도/수준 분석 | O | - |
| 문장 구조 패턴 참고 후 오리지널 작성 | O | - |
| 이미지/일러스트 직접 사용 | - | X (저작권) |
| 텍스트/문장 그대로 복사 | - | X (저작권) |
| 문제 유형 구조만 참고 | O | - |

**법적 안전선**: 교과서/교재의 **아이디어·구조·순서**는 저작권 보호 대상이 아님.
**표현물**(문장, 이미지, 레이아웃)은 보호 대상이므로 절대 직접 사용 금지.

---

## 2. 활용 영역별 작업 가이드

### 2-A. 커리큘럼 순서 검증 (즉시 활용, 최우선)

**목적**: 앱의 24유닛 파닉스 도입 순서가 교육학적으로 타당한지 검증

**작업 방법**:
1. 교과서의 파닉스 규칙 등장 순서를 `phonics-sequence-comparison.md`에 기록
2. 유명 파닉스 교재(Jolly Phonics, Oxford Phonics World 등)의 순서도 기록
3. 현재 앱 순서와 3열 비교표 완성
4. 불일치 항목에 대해 조정 여부 판단

**기대 효과**:
- B2G 납품 시 "교과 연계 커리큘럼" 주장 근거 확보
- 교사/학부모 신뢰도 향상
- 학습 효과 극대화 (검증된 순서 적용)

**관련 파일**: `phonics-sequence-comparison.md`

---

### 2-B. 단어 선정 대조 (즉시 활용)

**목적**: 교과서 필수어휘 vs 앱 수록어휘 교차 검증

**작업 방법**:
1. 교과서에서 학년별 필수어휘 목록 추출 → `textbook-words-3rd.md`, `textbook-words-4th.md`에 기록
2. `current-word-list.csv` (앱 현재 단어 목록, 자동 생성됨)와 대조
3. `word-alignment-checklist.md`에 분석 결과 기록:
   - 교과서에 있지만 앱에 없는 단어 → 추가 후보
   - 앱에만 있는 단어 → 유지 여부 판단
   - 양쪽 모두 있는 단어 → 핵심 어휘 확인

**기대 효과**:
- 교과서 필수어휘 커버율(%) 산출 → 마케팅 활용
- 불필요한 단어 제거로 학습 효율 향상

**관련 파일**: `current-word-list.csv`, `word-alignment-checklist.md`

---

### 2-C. 학년별 난이도 기준 설정 (중기 활용)

**목적**: 3학년 vs 4학년 콘텐츠 분리 기준 명확화

**작업 방법**:
1. 교과서 3학년 단원의 단어 수준(음절 수, 블렌드 포함 여부) 분석
2. 교과서 4학년 단원의 단어 수준 분석
3. 현재 `settings/page.tsx`의 `getMapping()` 학년별 유닛 배정이 적절한지 검증

**현재 매핑**:
| 학년 | 레벨 | 유닛 범위 |
|------|------|-----------|
| 1~2학년 | CoreA | unit_01~06 (Short Vowels) |
| 3학년 | CoreA | unit_01~12 (Short + Long Vowels) |
| 4학년 | CoreB | unit_01~18 (+ Blends & Digraphs) |

**검증 포인트**:
- 3학년 교과서에 이미 Blends(bl, cr 등)가 나온다면 → CoreB 일부를 3학년에도 열어줘야 함
- 4학년 교과서에 Diphthongs(oi, ou)가 안 나온다면 → unit_22까지 갈 필요 없음

---

### 2-D. microReading 문장 보강 (중기 활용)

**목적**: 각 유닛의 microReading 문장 품질 향상

**작업 방법**:
1. 교과서의 학년별 리딩 문장 길이·구조 분석 (단어 수, 문장 구조)
2. 수준에 맞는 **오리지널 문장** 작성 (교과서 문장 복사 절대 금지)
3. 현재 3문장 → 5문장으로 확대 검토

**현재 패턴**: `"A [noun]." → "A [adj] [noun]." → "[Subject] [verb] [prepositional phrase]."`
**교과서 참고 후 가능한 개선**: 의문문, 감탄문, 대화문 패턴 추가

---

### 2-E. 활동 유형 다양화 (장기 활용)

**목적**: Exit Ticket/퀴즈 유형을 교과서 활동 패턴 참고하여 확대

**교과서에서 참고할 활동 유형**:
| 교과서 활동 유형 | 앱 적용 방안 | 현재 구현 |
|-----------------|-------------|-----------|
| 듣고 고르기 | Sound Focus step 강화 | O (구현됨) |
| 빈칸 채우기 | Decode Words에 빈칸 모드 추가 | 부분적 |
| 그림-단어 매칭 | 이미지 에셋 확보 후 구현 | X (에셋 필요) |
| 따라 읽기 | Say & Check step | O (구현됨) |
| 문장 만들기 | Micro Reader 확장 | 부분적 |
| 알파벳 쓰기 | 터치 쓰기 모드 (향후) | X |

## 3. 작업 우선순위 (ROI 순)

| 순위 | 작업 | 소요 시간 | 영향도 | 비고 |
|:----:|------|:---------:|:------:|------|
| 1 | 단어 선정 대조 (2-B) | 2~3시간 | 높음 | 교과서 커버율 수치화 가능 |
| 2 | 커리큘럼 순서 검증 (2-A) | 1~2시간 | 높음 | B2G 납품 근거 |
| 3 | 학년별 난이도 검증 (2-C) | 1시간 | 중간 | getMapping() 조정 가능 |
| 4 | microReading 보강 (2-D) | 3~4시간 | 중간 | 콘텐츠 품질 향상 |
| 5 | 활동 유형 다양화 (2-E) | 장기 | 낮음 | 다음 버전에 반영 |

## 4. B2G 납품 시 활용 문구 (예시)

스캔 자료 분석 후 다음과 같은 문구를 제안서/교사 가이드에 사용 가능:

> "본 앱의 커리큘럼은 교육부 지정 초등 영어 교과서의 파닉스 도입 순서 및 필수어휘를 분석하여,
> 교과 진도에 맞춘 학습 경로를 제공합니다."

> "3~4학년 영어 교과서 필수어휘 커버율 XX% 달성"

(XX%는 단어 대조 후 실제 수치로 교체)
